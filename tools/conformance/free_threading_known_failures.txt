# DEADLOCK under monkey.patch -- NOT a runloom correctness bug; the documented
# foreign-thread / no-scheduler boundary.
#
#   test_func_annotations is the ONLY test_free_threading module that uses
#   concurrent.futures.ThreadPoolExecutor (every other module drives real
#   threading.Thread workers, which hit the foreign-thread-safe Co* Barrier and
#   PASS).  Under monkey.patch, ThreadPoolExecutor is replaced by
#   src/runloom/monkey/executors.py's CoThreadPoolExecutor, which "runs
#   submitted callables as FIBERS" -- and fibers only make progress under a
#   running runloom scheduler (runloom.run() / inside a goroutine).  The
#   conformance runner drives the stdlib test bodies from a bare main OS thread
#   with NO scheduler, so the submitted fibers can't all run; the test's
#   Barrier(NUM_THREADS=4).wait() then needs 4 concurrent participants that
#   never arrive -> deadlock.  Passes 2/2 WITHOUT monkey.patch (pure CPython
#   3.14t), verified this session.  This is exactly the "cooperative coordination
#   with a construct that needs true N-way concurrency deadlocks under the
#   cooperative model" limitation gated by RUNLOOM_STDLIB_REALTHREAD in
#   tests/_monkey_stdlib.py.
# =====================================================================
