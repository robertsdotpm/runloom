# Coverage exclusion manifest for the M:N scheduler + epoll netpoll.
#
# Each line: <fragment>  <start>-<end>  <CATEGORY>  <reason>
# Read by tools/cov_subsystem.py: excluded lines are removed from BOTH the total
# and covered counts, so the reported % is coverage of the COVERABLE surface.
# Every entry is a line that cannot be exercised by a clean-exiting test under
# this project's rules -- NOT a line we chose not to test.  Categories:
#   MIGRATION  - per-g-tstate / steal-woken scheduler, gated UNCONDITIONALLY on
#                RUNLOOM_ALLOW_UNSAFE_MIGRATION (a KNOWN-CRASH mode this project
#                forbids enabling; the gate is independent of hub count).
#   OOM        - calloc/PyMem_Calloc failure cleanup; the RUNLOOM_FAULT_* harness
#                has no allocator-failure site, so these need a malloc interposer.
#   SPAWNFAIL  - OS-thread / PyThreadState_New failure cleanup; no fault hook.
#   DEFENSIVE  - "can't happen" corruption/invariant guards (list-cycle, freed-or-
#                reused queue entry, lost-wake backstop) closed at the source.
#   PLATFORM   - needs a kernel/rlimit feature absent here (RLIM_INFINITY hard
#                limit, pre-4.5 kernel EPOLLEXCLUSIVE-EINVAL fold) or a backend not
#                compiled on Linux (the kqueue-only fd-registration bitmap).
#   CRASHONLY  - executes ONLY inside the fatal-signal crash handler, which dies
#                from the re-raised signal -> gcov never flushes those counters.
#   RACE       - a genuine free-threaded interleaving with no deterministic trigger
#                (a wake+re-park landing in a sub-microsecond lock-acquire window).
#
# (Reachable-but-not-yet-covered lines -- e.g. the idle-hub wakeup at
# init_fini 742-744, or the io_uring wake CAS path under-counted by a contended
# measure -- are deliberately NOT listed here.)
