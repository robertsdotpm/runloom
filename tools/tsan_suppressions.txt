# ThreadSanitizer suppressions for the runloom ext TSan runs (run_sanitizers_ext.sh).
#
# We instrument ONLY runloom's extension (-fsanitize=thread) and run it under a
# stock free-threaded CPython.  TSan therefore sees every access in runloom's C
# but NOT the interpreter's internal synchronization, so a handful of races
# *inside CPython itself* are reported that are not runloom bugs and that runloom
# cannot fix.  Suppress those by the CPython function/file they occur in.
#
# Keep this list MINIMAL and CPython-only.  Never suppress a frame in
# src/runloom_c/* -- those are real and must be fixed, not hidden.

# --- CPython free-threaded biased reference counting (Python/brc.c) ---
# Cross-thread refcount merge frees and re-mallocs _PyObjectStackChunk; the
# allocator's reuse synchronization is invisible to a half-instrumented build.
race:_Py_brc_merge_refcounts
race:merge_queued_objects
race:_PyObjectStack_Pop
race:_PyObjectStackChunk_New

# --- CPython interpreter/GC internals that touch shared objects ---
# (Add specific CPython functions here as broader suite runs surface them;
#  prefer the narrowest function name over a whole-file match.  Do NOT use a
#  blanket called_from_lib:libpython... rule -- every runloom race is "called
#  from" libpython via cfunction_call, so that would hide real bugs.)

# --- CPython non-atomic eval-frame hook setter (Python/pystate.c) ---
# _PyInterpreterState_SetEvalFrameFunc stores interp->eval_frame with a plain
# (non-atomic) write, and _PyEval_EvalFrame reads it with a plain load + indirect
# call.  runloom installs its RUNLOOM_PREEMPT eval-frame wrapper EXACTLY ONCE for
# the process lifetime (runloom_preempt_install; it used to toggle per
# mn_init/mn_fini, which raced a foreign thread's frame dispatch into a NULL
# call -> SIGSEGV; that toggle is gone).  The lone remaining NULL->hook publish
# still trips TSan against a foreign thread already dispatching frames, but it
# cannot crash (monotonic store to a non-NULL aligned pointer; the reader sees
# the old default or the new hook, both valid).  This is an upstream FT gap (the
# setter/reader would need to be atomic); an extension cannot close it
# (_PyEval_StopTheWorld is not exported, and an atomic store on our side does not
# make CPython's load atomic).  Suppressed on the CPython frame, not runloom's.
race:_PyInterpreterState_SetEvalFrameFunc
